When the ACE application is initialized, a HTML target element needs to be specified. In this exampe, the target element is the document's body. In it a div will be created, with the id: ace-app. This represents the main application container. ace-app has 2 children:

  • id: ace-middle-container, with the class ace-general-wrapper. This section contains:
    • An Optional Top Bar. Will receive the Id: ace-top-bar, and the class: ace-main-nav.
    • An Optional Side Menu. Will receive the class: ace-side-container.
    • The main content. Will receive the Id: ace-content, and the class: ace-content. Yes, the id and the class are the same.
  • id: ace-bottom-bar, with the class ace-footer. This is an optional section of the page, which, if it exists, it will be displayed at the bottom of the page.

If a Top Bar is to be created, a selector which will identify the custom top bar elements needs to be specified when the Application is initialized. The content of the selector will be removed from its original dom position, and appended to the Top Bar.

If an element with the class ace-side-menu-trigger is detected in the Top Bar, clicking this element will toggle the display of the side menu ( if one exists ).

If a Bottom Bar is to be created, a selector which will identify the custom bottom bar elements needs to be specified when the Application is initialized. The content of the selector will be removed from its original dom position, and appended to the Bottom Bar.

If a Side Menu is to be created, a selector which will identify the content of the side menu needs to be specified. The identified content will become the side menu itself( will receive the ace-side-container class).

The side menu should contain a user defined ACE accordion element, to which the class ace-side-menu should be given.

The accordion handler's function should have the name: switchMainMenuContent. This is the name of a predefined ACE function, which will automatically switch the visible content based on the selection in the accordion.

When the application is initialized, a selector needs to be specified, which will identify each distinct element that may be displayed in the main content area.

For example, for this example, the content selector used is this one: .ace-app-edit-windwow.

Each distinct element needs to also have an attribute defined with the name: tag. If the value of the tag attribute matches the tag attribute of one of the entries of the side panel accordion, whenever that element is selected, the content element will be displayed.

Each time a content element is displayed, a user defined method is called, if the method exist. The methods name is: onDisplayContentForTagXXXXX, where XXXXX should be replaced with the name of the tag that identifies the displayed content.

For each content element, an additional user method may be defined, with the name onBeforeDisplayContentForTag, where XXXXX should be replaced with the name of the tag that identifies the displayed content. If this method exists, and it returns false when ran, the content will not be changed.

To create the application, use the ace_app plugin method on the target element. The method accepts one parameter: an object, settings. through which the initialization of the application may be customized.

The settings object supports the following fields:

  • topbar: selector which will identify the content of the top bar; set an empty string to not display it.
  • sidemenu: selector which will identify side menu; set an empty string to not display it.
  • bottombar: selector which will identify the content of the bar bar; set an empty string to not display it.
  • contentselector: selector which will identify all the individual application content sections.
  • groupsidebarwithcontent: if true, the side menu and the application content will be group together in a div, identified with the id: ace-grouped-topside-container and the class ace-wrapper.

Please see the code in ace-init.js file for how this application is initiated.

The code in the ace-init.js file offers another method of switching content: through modifications of the page hash.

To use this method, simply create a HTML elements, and set their href attribute to '#CONTENT_TAG[/.../.../...]', where:

  • CONTENT_TAG: is the name of the tag which uniquelly identifies the content to be displayed.
  • [/.../.../...]: this is an optional content, a slash delimited string of characters, to further customize what actions should be taken once the content is displayed. This optional data will be found, together with the initial tag, in the parameter passed to the onDisplayContentForTagXXXXX user defined functions.
