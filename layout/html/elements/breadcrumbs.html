Elements of tip breadcrumbs may be used to display a location / position in a site, or in a wizard type form.

Forms will ignore breadcrumbs.

Ex1 - Wizard Type Breadcrumbs

  • Breadcrumbs of type Wizard can be used to guide the user through a set of given steps.
  • From a given position the user may navigate to any of the previous steps.
  • but to go forword, the user needs to perform some sort of verification / validation procedure.
  • In this case, this is acomplished by pressing the move forword button.
  • Every time a a new step is activated, a click event is triggered, which can be interecepted.

Ex2 - Dynamic Breadcrumbs, wiht renderer

  • In this example, the breadcrumb starts without any data.
  • Only the home step is displayed. This step is not part of the breadcrumb's dataset.
  • Only one step is the active one, and that is the latest step added.
  • Every time a previous step is activated, all other newer steps are deleted.
  • This type of behaviour may be useful in cases where multiple selections or scenarios exist.
  • A renderer function has been designed in this case, which colors the text of each step in red or green.

Ex 3 - Visibility

  • Depending on the situation, some steps may be made visible, or invisible.
  • The standard move next, or move previous functionality will ignore the invisible steps, and simple jump over them.
  • In this example all steps are clickable (except the home), and they are not deleted.
  • Use the Toggle Visibility button to change the visibialy of some fields.
