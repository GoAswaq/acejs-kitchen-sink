Elements of tip multicheck are used to select one or more elements from a list of elements.

Multicheck fields are using arrays of data as a datasource, in which each element in the array is an object, with one or more fields:

One of these fields will be used to determine the TEXT which is displayed (default field: name), and another field is chosen to become the actual VALUE (default field: value) of the current selection

The multicheck may be give a set of static data, or it may be configured to load it's own data from the server automatiaclly.

If the element is in a form, and it is porperly configured, the form will automatically retrive its value on the save operation, if the multicheck has been changed.

When displaying the selector box the style of the selection columns may be changed through the *customtoptionsselectionclasses* setting, through which the desired CSS class may be set. For example, by using the ace-col-6 class, the number of colums will be 2, and through ace-col-4, the number of columns will be 3.

Ex. 3 - Multicheck in a Form

  • in this example, there are several ways presented in which which the initial selection of a multicheck may be set when in a form
  • the multicheck has the fieldname selected_students
    1. no selection - the multicheck will be displayed without a selection; selected_students will be given an empty string as value
    2. object - selected_students will be given an object, in which the every property name is a selection id, and their value their display values
    3. json - selected_students will be given a string, representing the JSON encrypted format of an object, in which the every property name is a selection id, and their value their display values
    4. single selection - selected_students will be given a single value, representing the id of 1 selected item; as display text, the same id will be displayed; when the selection window is opened, if the id is part of the loaded data, the coresponding entry will be selected in the window
    5. single selection with explicit text - selected_students will be given an empty string; the value of the multicheck will be set AFTER the record has been loaded, using the ace value function, where the display text will be given in the extra parameter
