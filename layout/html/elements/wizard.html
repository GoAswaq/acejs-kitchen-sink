ACE fields of type wizard are used to create complex elements, which are generelley known as wizards.

They combined together multiple forms, buttons, and progress bars.

Each step works as a distinct form. All step forms share the same record.

Calling ACE value method returns or set the working record.

Ex. 1 - Example

  • This wizard here gives an example for:
    • Step 1: steps with auto field validation.
    • Step 2: steps without auto field validation.
    • Step 3: steps with custom user logic validation.
    • Step 4: steps with server side communication and validation; Attention: the step is configures so that the server validation takes place on BOTH forword, and backwords navigation.
    • Step 5,6,7: these display the step visibility setting, and how it can be changed.
    • Step 8: this step doesn't save its data! Write anything, and the field values will remain as they are. Also, the form loads some data from the server as well!
    • Step 9: the final step, with custom user callback.
