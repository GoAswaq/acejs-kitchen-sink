Elements of tip text are used to display an editable text, and to retrieve the typed words.

They consist of two main components:

  • a label text
  • an input element

If the element is in a form, and it is porperly configured, the form will automatically retrive its value on the save operation, if the text has been changed.

Ex. 1 - Simple Text Element

  • This example contains two text fields, with the label having their implicit position on top.
  • On of the fields has a placeholder, the other does not.
  • If the field doesn't have a placeholder, or a text value given, then the top label falls down, and functions as a palceholder

Ex. 2 - Label align, custom label sufix, and field modifications.

  • The label of a field may be modified from: top, to left.
  • Also, the palceholder may be removed, or added, as needed.
  • Every label text has by default the character : added as a sufix. This sufix can be customized.

Ex. 3 - onchange event example

  • the field generates a change event each time the field loses input focus, or when the ENTER key is pressed, IF the current text value is different than the previous one.

Ex. 4 - Badge

  • small bits of text may be configured to be displayed by a text field, in addition to it's usual component.
  • these small bits of text are called: badges

Ex. 5 - Fields with initial value, and readonly

  • a text field may have an initial value set on creation, like in this example
  • a field may preveneted from being modified by users through the configuration setting readonly
