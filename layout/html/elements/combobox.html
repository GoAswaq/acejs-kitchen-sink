Elements of tip combobox are used to display a select element, with customized data.

Comboboxes are using arrays of data as a datasource, in which each element in the array is an object, with one or more fields:

One of these fields will be used to determine the TEXT which is displayed, and another field is chosen to become the actual VALUE of the current selection

The combobox may be give a set of static data, or it may be configured to load it's own data from the server automatiaclly.

If the element is in a form, and it is porperly configured, the form will automatically retrive its value on the save operation, if the combobox has been changed.

Ex. 3 - Dynamic Combobox - setData

  • Initially, the combobox does not have any items.
  • To add a new select option, write something, and press enter into the text element bellow.
  • when the combobox is readonly, its value may not be modified anymore through the GUI
  • If the text has not already been added to the combobox, it will.
  • A new element will be added to the combobox through the setData method.
  • For this example, the value of the selected item will be the same as the value of the entered text.
  • If the checkbox autoswitch is checked, each time a new text is added, the combobbox will automatically swith to the new element.
  • Even if the autoswitch is not-checked, when the first element is added, the checkbox will switch to that value.

Ex. 5 - Combobox, with data from Server + renderer

  • Initially, the combobox does not have any items.
  • Press the Load button, and the combobox will send am ajax request to the server, to retrive some data.
  • The data retrieved needs to have a specific format, encoded in a json.
  • Each loaded element has several fields, in addition to the name and id field.
  • Use getRecord to retrieve all of them.
  • A renderer function has been used which displayed additional stundent information.
  • Please note that a loading animation appears over the combobox for the duration of the server request.
