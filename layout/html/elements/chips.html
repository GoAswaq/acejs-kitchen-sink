Elements of tip chips help users enter a multiple selection from an unspecified set.

The user may enter a text, press enter, and the text will be added as a new chip to the selection.

Chips may be deleted by pressing their X button

Chips don't have a label component

Forms ignore chips.

Ex. 1 - The simples chip element

  • By default, the chip does not allow duplicates.
  • Each chip added receives an unique id, within the dataset of the chip.
  • The default max number of chips is 100.

Ex. 2 - Other functionality

  • This chip allows duplicate texts.
  • The maximum chips number is set to 5.
  • The chips already has a set of data on creation.
  • The chip may have a custom method specified, which validates the input field. In this case, only short text are allowed to be added as chips: max length of 3 characters.
  • Two events are generated each time a new chip is added, or removed. This events may be intercepted.
  • The chip may have a button displayed, which, when pressed, will clear the contents.
  • Also, if desired, clicking anywhere on a chip can remove the chip. By default, chips may be removed only when the X is pressed.

Ex. 3 - Chips with combobox selector.

  • The user may type the texts manually, or the texts may be chosen from a combobox.
  • Once an element is selected from the combobox, that element will no longer appear there.
