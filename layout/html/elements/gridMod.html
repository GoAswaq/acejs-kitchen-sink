These are additional classes, which when applied over specific ACE fields, it modified the way they look, or behave.

Ex. 1 - Table Grid Modifier Classes

  • These classes modify the appearance or functionality of table grids:
  • The following classes modify the height of the grid. All of these decrease the given amount from the maximum height of the grid:
    • ace-grid-height-minus-250,ace-grid-height-minus-230,ace-grid-height-minus-200
    • ace-grid-height-minus-150,ace-grid-height-minus-120,ace-grid-height-minus-110,ace-grid-height-minus-100
    • ace-grid-height-minus-80,ace-grid-height-minus-60,ace-grid-height-minus-50
  • The following classes modify the height of the internal body(the section containing the rows) of the grid. All of these decrease the given amount from the maximum height of the body:
    • ace-grd-body-height-minus-250,ace-grd-body-height-minus-230,ace-grd-body-height-minus-200
    • ace-grd-body-height-minus-150,ace-grd-body-height-minus-120,ace-grd-body-height-minus-110,ace-grd-body-height-minus-100
    • ace-grd-body-height-minus-80,ace-grd-body-height-minus-60,ace-grd-body-height-minus-50
  • ace-grid-hide-header: the class hides the column headers, and the totals row.
  • ace-grid-hide-toolbar: the class hides the toolbar of the grid
  • ace-grid-only-row: the class hides the footer, the toolbars, and any br tags present in the body of the grid
  • ace-minimal-grid: the class hides the toolbar, the header, and the footer totals.
  • ace-compact-grid: lowers the padding and hight of all rows and cells in the grid
  • ace-thin-row: as oposed to all the other classes here, this one must be applied as an additional class on the column descriptor, and it affects only the cells belonging to those columns. It makes the cells thiner. In the example bellow the class is added to all user defined columns.
  • ace-thin-rows-grid: do not confuse it with ace-thin-row. This one makes the grid more compact than ace-compact-gird. Can be combined with: ace-thin-rows-grid-with-header to keep the header visible.
  • ace-flex-grid: If no internal height classes are defined, make it so that the footer will be displayed immidietly after the last row. Also, it can be easily combined with classes that modify the max internal height of the grid. Examples of classes that set the max internal height of the grid with a FIXED amount in pixels:
    • ace-flex-grid-height-600
    • ace-flex-grid-height-400
    • ace-flex-grid-height-300
    • ace-flex-grid-height-250
    • ace-flex-grid-height-200
  • ace-responsive-grid: This class is activated at resolutions bellow 680px. It hides the header, and on each row, for each column it displayes the column header text in front of the displayed value.
  • ace-grid-absolute-footer: Makes it so that the footer, if visible, will be displayed at the bottom of the grid container's element.
  • ace-white-space-normal:This is NOT a grid specific class, but a global text related one. Use this class to force the grid to display the entire text of its cells, even thouhg this might lead to thicker rows.

Ex. 2 - Panel Grid Modifier Classes

  • These classes modify the appearance or functionality of panel grids.
  • These classes change how many row panels will be displayed on one row:
    • ace-grid-cols-3: 3 panels will be displayed on one row.
    • ace-grid-cols-2: 2 panels will be displayed on one row.
    • ace-grid-cols-4: 4 panels will be displayed on one row.
    • ace-grid-cols-6: 6 panels will be displayed on one row.
    • ace-grid-cols-auto: auto determined by the hight of each panel.
  • ace-compact-row-panel: makes the panel more compact, by making the default paddings between some elements smaller.
