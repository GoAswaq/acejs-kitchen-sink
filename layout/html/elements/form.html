Elements of tip form containers: they can contain and display more ACE fields.

If the ACE fields are configured corectly, the form will know how to properly display a set of data.

The forms iterates through all fields of the data set, and attempts to match them to an existing ACE field.

If such a field is found, the data from the field will be passed to the ACE field.

The matching is done through the property fieldname of the ACE fields, which needs to corespond to a field property name from a dataset

It is recomended that there be no multiple fields with the same fieldname in the same form

The form is also able to gather the information which has changed, and send it over to the server, or to another operation for processing

The form knows how to work automatically with most ACE fields of Data Input type, except chips.

There are two major types of forms:

  • popup - the form will be displayed as a hovering window OVER all other existing window content.
  • custom - the form will be displayed differntly, its exact method being left to the programmer to specify

All forms must have a template specified: a template in this case represents the ID of a DIV, which will be used to create the contents of the form itself.

Because the content of a template is cloned each time the template is used, all elements from a template should not have an id property defined.

This field is created explictdly by the form during its creation process, via the method onninit. All autocomplete fields that appear in templates should be created in this manner, to ensure there are no problems.

Ex. 4 - The same as Ex.3, with a few specialized classes

  • The default ace css contains a few classes which help display forms in certain ways
  • This classes need to be added manually where needed to the div in which the form will be rendered
  • When this class combinations are used, the form will be rendered OVER the existing content, without disturbing it.
  • This classes are (the top, bottom, and side indications are relative to the container in which the form finds itself):
    • ace-independent-form - to display the form from the top to bottom, using it's full width.
    • ace-independent-form ace-lateral-form - to display the form from the top to bottom, on the side
    • ace-advanced-panel from-top - the form will be displayed at the top
    • ace-advanced-panel from-bottom - the form will be displayed at the bottom
  • Chose from the combobox bellow how the form should be displayed.
  • In this example, the form is moved right before creation from its current location to the page body.
