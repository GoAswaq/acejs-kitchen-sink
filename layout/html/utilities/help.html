There are a few ways in which various help information may be included in the page.

See the examples bellow for details

Ex. 1 - Tooltips

  • Many ACE fields support tooltips.
  • A tooltip represents an explanation text that will appear when the mouse hovers over an ACE field.
  • The text in this case should be short and streight to the point.
  • To specify a tooltip, the tooltip parameter should be used when creating or modifying an ace field.
  • These tooltips are displayed using native browser support.
  • Hover the mouse over the fields bellow to see the tooltips.

Ex. 2 - Hints

  • ACE fields may have hints.
  • Hints are a way to display in the page *detailed* information about a field. The information may contain large amounts of text, images, and/or videos.
  • The way this works, is that the DOM element that contains the explanation will be made visible when the ACE field receies input focus. So only ACE fields that may receive input focus will trigger this behaviour.
  • To specify a hint, the property hintselector of an ACE item must be set. The hintselector is a CSS selector pointing at the DOM element that contains the explanation
  • In addition to this, the explanation elements should have the class ace-hint.
  • By default, when a hint is displayed, all other hints next to it will be hidden.
  • The HINT area should display an closing element which will make the hint area hidden.

Ex. 3 - Information Badges

  • All ACE fields support information badges.
  • Information badges come in two types:.
    1. links: when clicked, a new broswer tab will be opend to the specified link; badge icon:
    2. texts: when clicked, the explanation text will be displayed in a popup window; badge icon:
  • A field may have on type, both types, or none
  • To specify a link badge, set the the infolink parameter to the desired URL
  • To specify a text badge, set the the infotext parameter to the desired text.
  • By default, the badges are positioned to the left. To position them to the right of the field, set the parameter infopos to right. Defualt value: left.
  • By default, the badges are shown all the time. To only show them on hover, set the parameter infohover to true.
  • To remove a badge, set its related parameter to an empty string or false.
