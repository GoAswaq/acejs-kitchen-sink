All ACE fields can validate themselves.

Through validation, the ACE fields compare their current value to a set conditions.

If all the conditions are met, the field is valid, otherwise it is invalid, and a warning icon, with a text will be displayed.

Ex. 1 - Validations

  • Each field can display a warning in the form of an asterix, notifying the user that the value within will be validated.
  • After displaying the warning, on the next validation attempt, the invalid fields will be highlighted in some obvious way, and a warning icon will be displayed.
  • If the warning icon is clicked, a toast message will appear, that will contain a clue as to why the field value was not validated.
  • The fields know automatically to verify the following conditions
    • minval: the value of the field needs to be numerical, and greater or equal than the specified amount.
    • maxval: the value of the field needs to be numerical, and less or equal than the specified amount.
    • minlength: the value of the field needs to be a string, whose lenght is at least the specified amount.
    • maxlength: the value of the field needs to be a string, whose lenght is at least the specified amount.
    • allowempty: something, anything, needs to be written inside.
    • customvalidation: a function, or the name of a function in which custom validation logic may be implemented.
  • Other validation modifiers:
    • onlywhenvisible: the validation will be triggered only when the field is visible.
    • forceerror: if true, the field will never display the warning asterix, but will instead display the full error notification.
